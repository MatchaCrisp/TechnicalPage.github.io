Introduction

Your HTML webpage does not have to be static! CSS has built-in animation capabilities. By combining them with your HTML5 code, you can make your webpage come to life.

But before we dive in, here's a quick summary on animations in general: Animations are made of a series of still pictures, or frames, each slightly different from the last. Key frames are the bare minimum number of frames that still tells the same story as the full animation itself. The rest of the frames between key frames are called in-betweens, and they aid in the smooth transition from one key frame to the next.

The animation of a ball bouncing across the screen was conceptualized with a dotted line tracing its movement in 1. Then all the extreme positions are drawn in 2, these are the keyframes. Finally the in-betweens come in and flesh out what the animation should look like between the keyframes.

CSS animations

When animating using CSS, you only need to define the key frames of your animation, and CSS will do the in-between frames for you. This feature takes a little to get used to, but is very worth the effort, as it saves countless hours of manually filling in the in-betweens. The caveat is the algorithm CSS uses to fill in the blank may not be what you expect, so tweak around with your key frames until CSS understands exactly what you want.

You can define those animation key frames by using the @keyframes CSS rule. First declare the rule with a descriptive animation name, then add keyframes by using percentages and a set of curly brackets. The percentages represent the percent progress in the animation, and the curly brackets will contain what you want the animated element to look like, described in CSS properties.

Finally, you tell the browser which element should be animated by using the animation-name property. You add the property to the selector of the element you want animated, and the property value is the descriptive name you gave your @keyframes rule.

Animation-duration

The animation-duration property defines the duration of one animation cycle, from 0% to 100%. For instance giving the property a value of 4s means one playthrough of the animation will take 4 seconds.

The property can accept positive time values with the units:

  • s (seconds)
  • ms (milliseconds)

CSS uses a default value of 0s when no duration is specified, indicating that no animation should occur.

Animation-iteration-count

The animation-iteration-count property defines how many times the animation plays before stopping.

The property can accept the following as input values:

  • infinite (animation runs until page is closed)
  • positive decimal number (a decimal value means the animation runs for a fraction of the total duration)

CSS uses a default value of 1 when no iteration count is specified.

Animation-delay

The animation-delay property defines the delay for the start of the animation.

The property can accept time values with the units:

  • s (seconds)
  • ms (milliseconds)

CSS uses a default value of 0s when no delay is specified.

a negative time value tells the browser to immediately play the animation from x point onwards.

Animation-direction

The animation-direction property defines the direction of play of the animation.

The property can accept the following as input values:

  • normal (default)
  • reverse
  • alternate (first normal then reverse)
  • alternate-reverse (first reverse then normal)

Animation-fill-mode

The animation-fill-mode property defines how the element containing the animation behaves when the animation is not playing.

The property can accept the following as input values:

  • none (default)
  • forwards (container displays last keyframe executed when animation ends)
  • backwards (container displays first keyframe executed before animation begins)
  • both(both forwards and backwards)

Animation-play-state

The animation-play-state property tells the animated element if the animation should be running or paused.

The property can accept the following as input values:

  • running (default)
  • paused

note the lack of stop as a value.

Animation-timing-function

the animation-timing-function property defines how the animation is played over the duration of the animation.

The property can accept the following as input values:

  • ease (default)
  • linear
  • ease-in
  • ease-out
  • ease-in-out

When these pre-defined speeds are not what you want, you can specify your own using the cubic-bezier function.

Bezier curve

The cubic-bezier function provides a way to custom design an animation timing function. On a 1 by 1 xy plane, where x represents the % animation duration, and y represents the % animation progress, a Bezier curve is a cubic function that represents the animation progress over time. It does so by defining 4 points, P0, P1, P2, and P3. See below for a visual representation of a Cubic-Bezier curve:

P0 and P3 are fixed at (0,0) and (1,1) respectively. The Bezier curve is customized by modifying P1 and P2.

Syntax: animation-timing-function:cubic-bezier(P1x,P1y,x2,y2)

where (x1,y1) and (x2,y2) are coordinates for P1 P2 respectively.

Note how P1 and P2 are not on the curve directly, instead they serve as points that skews the curve.

Visit cubic-bezier.com by Lea Verou to see cubic-bezier in action, and to make some cubic-beziers yourself.

animation shorthand

The animation shorthand property provides a quick way to set all of the animation properties mentioned above in one line.

Syntax: animation: <name> <duration> <timing-function> <delay> <iteration-count> <direction> <fill-mode> <play-state>

Being able to assign values to all 8 CSS properties in one makes for more compact and readable code.
